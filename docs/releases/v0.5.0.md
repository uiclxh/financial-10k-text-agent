# v0.5.0 Step 14.1 - Artifact-Aware Orchestrator

Release date: 2026-05-21

Orchestrator release for resumable local pipeline execution. This version lets
the `run` command execute downstream stages from standard run-directory
artifacts instead of only initializing a workspace.

## Changes

- Added artifact-aware execution through the run controller.
- Preserved initialization-only behavior when execution is not requested.
- Added resumable execution for:
  - labels
  - rolling splits
  - text features
  - model training
  - evaluation and event-based backtest
  - audit
  - report
- Added stage skipping when outputs already exist.
- Added stage blocking when required upstream artifacts are missing.
- Added `orchestrator_report.json` with:
  - stage status
  - missing inputs
  - generated outputs
  - stage metrics
  - final run status
  - blocked reason
- Updated `pipeline_contract.json` to document the artifact-aware controller
  and standard run-directory inputs.

## Boundaries

This release expects standard local artifacts such as `document_manifest.jsonl`,
`prices.csv`, and `parsed_sections.jsonl` when upstream stages need to run.
Config-level raw data paths and deployment packaging are completed in `v0.6.0`.
