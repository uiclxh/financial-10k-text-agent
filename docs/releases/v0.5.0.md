# v0.5.0 Step 14.1 - Artifact-Aware Orchestrator

Release date: 2026-05-21

This release addresses the largest remaining engineering gap after Step 13: the
`run` command can now execute a resumable pipeline controller instead of only
initializing a run workspace.

## Phase

Step 14.1:

- Artifact-aware run orchestration
- Standard run-directory artifact conventions
- Missing-input blocking report
- End-to-end continuation from existing artifacts

## Deployment Placement

Local first, with GitHub storing the controller code, tests, and small smoke
fixtures.

## Why This Phase Belongs Here

Formal 10-K experiments still depend on local data permissions, SEC documents,
price files, and large generated artifacts. The orchestrator therefore works
against standard local run-dir artifacts and records what it can execute, skip,
or block.

## Included Capabilities

- `python -m text_factor_lab run --config <path> --execute`.
- Initialization-only behavior remains available without `--execute`.
- Resumable execution for:
  - labels
  - rolling splits
  - text features
  - model training
  - evaluation and event-based backtest
  - audit
  - report
- Stage skipping when outputs already exist.
- Stage blocking when required upstream artifacts are missing.
- `orchestrator_report.json` with:
  - stage status
  - missing inputs
  - generated outputs
  - stage metrics
  - final run status
  - blocked reason
- `pipeline_contract.json` now documents the artifact-aware controller and the
  standard run-dir inputs.

## Main CLI

```bash
python -m text_factor_lab run \
  --config configs/text_factor_lab/mvp_v0.yaml \
  --execute
```

Allow diagnostic report generation if audit fails:

```bash
python -m text_factor_lab run \
  --config configs/text_factor_lab/mvp_v0.yaml \
  --execute \
  --allow-failed-audit-report
```

## Boundaries

This is not yet raw-data orchestration from zero. The controller expects standard
run-dir artifacts such as `document_manifest.jsonl`, `prices.csv`, and
`parsed_sections.jsonl` when those upstream stages need to run. Future work
should add config-level raw data paths, SEC download scheduling, and richer
deployment packaging.
