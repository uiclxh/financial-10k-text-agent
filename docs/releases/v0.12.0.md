# v0.12.0 Research Universe

Release date: 2026-05-23

Universe-layer upgrade for survivorship-bias control and research-grade data
contracts. This release keeps the original lightweight `tickers_file` path for
MVP runs while adding explicit schemas and quality checks for a security master,
dated universe membership, and entity link history.

## Changes

- Added `SecurityMasterRecord`.
- Added `UniverseMembershipRecord`.
- Added `EntityLinkHistoryRecord`.
- Added optional config paths:
  - `universe.security_master_file`
  - `universe.membership_file`
  - `universe.entity_link_history_file`
- Added `universe.universe_data_level`.
- Added CSV loaders for security master rows.
- Added CSV loaders for universe membership intervals.
- Added CSV loaders for entity link history.
- Extended `universe_quality_report.json` with:
  - security master row count
  - membership row count
  - entity link row count
  - membership rows missing security master links
  - membership rows missing entity link history
  - low-confidence link rows
  - membership market-cap missing rows
  - delisted membership rows
- Added formal blockers for missing research-grade tables.
- Added formal blockers for membership rows that cannot be joined to security
  master or entity link history.
- Added research-grade universe example CSV files.
- Added `configs/universe/universe_research_grade.yaml`.
- Added tests for research-grade table loading and quality gates.

## Boundaries

This release defines and validates the research-grade universe contract. The
included CSVs are small fixtures, not a licensed CRSP/Compustat/WRDS universe.
Formal empirical work still requires populating these tables with survivorship-
free historical data, delisting information, dated market caps, and reliable
security identifiers.
