# v0.4.0 Step 13 - Report Agent

Release date: 2026-05-20

This release adds the first executable Report Agent for completed MVP runs. It
turns audited model, evaluation, and backtest artifacts into a reproducible
Markdown report and a machine-readable JSON summary.

## Phase

Step 13:

- Report generation
- Report summary artifact
- Formal/exploratory conclusion gate
- CLI polish for the report stage

## Deployment Placement

Local deployment first, with GitHub storing code, tests, sample fixtures, and
release documentation.

## Why This Phase Belongs Here

Reports should be generated next to the run artifacts because formal experiments
can contain large local outputs and licensed data references. GitHub should keep
the deterministic report generator and tests, while large run directories remain
outside the repository.

## Included Capabilities

- New `text_factor_lab.reports` package.
- `python -m text_factor_lab report` CLI command.
- `report.md` output.
- `report_summary.json` output.
- Executive summary, research question, data/sample, label, feature, model,
  prediction, backtest, robustness, audit, failure, conclusion, and reproducible
  command sections.
- Automatic best prediction metric summary.
- Automatic top backtest summary.
- Audit failure handling:
  - `audit_status=fail` blocks standard report generation.
  - `--allow-failed-audit` permits diagnostic-only reports.
- Run status update to `reported` when a pass/warn audit report is generated.

## Main CLI

```bash
python -m text_factor_lab report \
  --run-id tflab_10k_mvp_v0_001 \
  --run-dir runs/text_factor_lab/tflab_10k_mvp_v0_001
```

Optional diagnostic report for failed audits:

```bash
python -m text_factor_lab report \
  --run-id tflab_10k_mvp_v0_001 \
  --run-dir runs/text_factor_lab/tflab_10k_mvp_v0_001 \
  --allow-failed-audit
```
