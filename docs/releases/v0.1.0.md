# v0.1.0 Phase 1 - Foundation / Project Initialization

Release date: 2026-05-20

This release marks the first engineering phase of `Financial 10-K Text Agent`.
It packages the lightweight project foundation that should live both locally and
on GitHub for version control, review, and CI readiness.

## Phase

Steps 1-3:

- Project initialization
- Core schema layer
- Orchestrator / run manager scaffold

## Deployment Placement

Local + GitHub.

## Why This Phase Belongs Here

The code is light, stable, and suitable for early version control. GitHub is
useful at this stage for tracking schema changes, reviewing configuration
contracts, and preparing CI before larger data artifacts enter the workflow.

## Included Capabilities

- Python package scaffold under `src/text_factor_lab`.
- CLI entry point through `python -m text_factor_lab`.
- `mvp_v0.yaml` experiment config scaffold.
- Pydantic config and artifact schemas.
- `formal_run` / `exploratory_run` base concepts.
- Run directory creation.
- `config_snapshot.yaml`.
- `run_status.json`.
- `failure_log.jsonl`.
- Initial orchestrator contract.

## Acceptance

```bash
python -m text_factor_lab run --config configs/text_factor_lab/mvp_v0.yaml
python -m pytest
python -m ruff check .
```

## Boundaries

This phase does not include SEC data ingestion, parsing, labels, features,
models, backtesting, or audit logic. Those are separated into later releases so
the repository history stays understandable.
