# v0.1.0 Foundation Release

[中文](#中文) | [English](#english)

## 中文

发布日期：2026-05-20

这是 `Financial 10-K Text Agent` 的第一版 foundation release，覆盖从零开始到第八步的工程基础。当前版本的目标不是完成完整因子研究闭环，而是把后续研究所需的配置、schema、数据、解析、标签和切分基础打稳。

### 已完成步骤

1. 项目脚手架
2. Config 与 schema 层
3. Orchestrator / run manager
4. Universe manifest 与质量报告
5. SEC EDGAR 10-K filing metadata
6. SEC 10-K section parser
7. Price data 与 label builder
8. Rolling year split 与防泄漏检查

### 关键能力

- `formal_run / exploratory_run` 配置基础
- Pydantic schema validation
- run status 与 failure log
- fixed universe manifest validation
- SEC EDGAR CIK、submissions、Archives URL 工具
- SEC acceptance time 转 UTC
- 10-K `Item 1 / 1A / 3 / 7` section extraction
- table of contents heading 误判过滤
- section char span 与 SHA256 hash
- price CSV loader
- `log_return` 与 `simple_return`
- `CAR_1_n`
- `realized_volatility_1_n`
- `realized_volatility_annualized_1_n`
- explicit label failure report
- expanding train / rolling validation / rolling test split
- embargo leakage report

### CLI

```bash
python -m text_factor_lab run --config configs/text_factor_lab/mvp_v0.yaml
python -m text_factor_lab parse-10k --manifest-record path/to/record.json --document path/to/10k.html --output-dir runs/example/parsed
python -m text_factor_lab build-labels --document-manifest path/to/document_manifest.jsonl --prices path/to/prices.csv --labels-output runs/example/labels.jsonl --failures-output runs/example/label_failures.jsonl --target CAR_1_20 --target realized_volatility_1_20
python -m text_factor_lab build-splits --labels runs/example/labels.jsonl --assignments-output runs/example/split_assignments.jsonl --leakage-output runs/example/split_leakage.jsonl --sample-start 2010-01-01 --sample-end 2024-12-31 --train-years-min 5 --validation-years 1 --test-years 1 --embargo-days 20
```

## English

Release date: 2026-05-20

This is the first foundation release of `Financial 10-K Text Agent`. It covers the first eight implementation steps from a blank workspace to a reproducible engineering base. The goal of this release is not to finish the complete factor research loop, but to stabilize the config, schema, data, parsing, label, and split layers needed for later research modules.

### Completed Steps

1. Project scaffold
2. Config and schema layer
3. Orchestrator / run manager
4. Universe manifest and quality report
5. SEC EDGAR 10-K filing metadata
6. SEC 10-K section parser
7. Price data and label builder
8. Rolling year split and leakage checks

### Key Capabilities

- `formal_run / exploratory_run` configuration base
- Pydantic schema validation
- run status and failure log
- fixed universe manifest validation
- SEC EDGAR CIK, submissions, and Archives URL helpers
- SEC acceptance time conversion to UTC
- 10-K `Item 1 / 1A / 3 / 7` section extraction
- table-of-contents heading false-positive filtering
- section char spans and SHA256 hashes
- price CSV loader
- `log_return` and `simple_return`
- `CAR_1_n`
- `realized_volatility_1_n`
- `realized_volatility_annualized_1_n`
- explicit label failure report
- expanding train / rolling validation / rolling test split
- embargo leakage report
