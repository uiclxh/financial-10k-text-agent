# v0.14.0 - Portfolio Accounting

v0.14.0 upgrades the daily portfolio layer from fixed rebalance weights to
drifted daily position accounting.

## Changes

- Adds `position_accounting` to `portfolio_returns.jsonl`.
- Adds ending exposure fields to each portfolio return record:
  - `ending_long_exposure`
  - `ending_short_exposure`
  - `ending_gross_exposure`
  - `ending_net_exposure`
- Computes daily price-panel returns using beginning-of-day weights.
- Drifts position weights after each holding-day return, so later days no
  longer assume fixed `+1 / -1` exposure.
- Keeps transaction cost on the rebalance day and zero daily turnover while the
  same basket is held.
- Preserves the old `label_window` fallback for runs without a price panel.
- Reports portfolio position-accounting methods in the generated reports.

## Why It Matters

This makes the portfolio artifact closer to research-grade accounting. Daily
returns now reflect holding-period weight drift instead of repeatedly applying
the original rebalance weights.
