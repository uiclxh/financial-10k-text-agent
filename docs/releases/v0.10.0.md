# v0.10.0 Multiple Testing

Release date: 2026-05-23

Statistical inference upgrade for reducing data-snooping risk. This release
turns tested-specification disclosure into concrete machine-readable artifacts
and adds first-pass multiple-testing adjustment for the core research metrics.

## Changes

- Added tested-specification schema.
- Added multiple-testing family schema.
- Added multiple-testing report schema.
- Added `text_factor_lab.inference`.
- Added `tested_specifications.jsonl`.
- Added `multiple_testing_report.json`.
- Added specification records for:
  - out-of-sample Rank IC
  - event-level Newey-West t-stat
  - portfolio Sharpe diagnostics
- Added Bonferroni adjusted p-values.
- Added Holm adjusted p-values.
- Added Benjamini-Hochberg FDR adjusted p-values.
- Grouped specifications by target and metric family.
- Integrated inference artifacts into `evaluate-models`.
- Integrated inference artifacts into `run --execute`.
- Added audit validation that tested-specification rows and the multiple-testing
  report are present and aligned.
- Added report section summarizing adjusted p-values and discovery counts.
- Added tests for inference generation, JSON artifact round trips, audit
  acceptance, and e2e smoke output.

## Boundaries

This release implements the first research-grade multiple-testing layer. It does
not yet implement Deflated Sharpe Ratio, Probability of Backtest Overfitting,
Combinatorial Purged Cross-Validation, bootstrap confidence intervals, or
clustered standard errors.
