# v0.2.0 Phase 2 - Data, Parsing, Labels, Splits, Features

Release date: 2026-05-20

This release packages the data-heavy research preparation phase. It covers the
parts that are best debugged locally first because SEC documents, price data,
file paths, timestamps, and artifact permissions are more complex than the base
application scaffold.

## Phase

Steps 4-8 plus the text feature layer:

- Fixed universe manifest
- SEC EDGAR 10-K metadata
- SEC 10-K parsing
- Price data and label construction
- Rolling splits and leakage checks
- Dictionary tone and train-window TF-IDF features

## Deployment Placement

Local-first, then GitHub for code, schemas, tests, and small fixtures.

## Why This Phase Belongs Here

Data paths, file sizes, source permissions, SEC timestamps, and parsing edge
cases are easier to debug locally. GitHub should receive the reproducible code,
schema definitions, sample fixtures, and tests, while large or licensed data
should remain outside the repository.

## Included Capabilities

- Fixed universe validation with ticker, CIK, sector, industry, mapping source,
  and delisting fields.
- SEC EDGAR CIK normalization, submissions helpers, archive URL helpers, and
  10-K metadata handling.
- 10-K parser for Item 1, Item 1A, Item 3, and Item 7.
- Table-of-contents false-positive filtering.
- Section span and SHA256 hash artifacts.
- Price CSV loader.
- `log_return` and `simple_return`.
- `CAR_1_n`.
- `realized_volatility_1_n`.
- `realized_volatility_annualized_1_n`.
- Explicit label failure records.
- Rolling train / validation / test splits.
- Embargo leakage reports.
- Dictionary tone features.
- Section-level and full-text TF-IDF features fit on train windows only.
- `feature_manifest.json` and `vocabulary.json`.

## Main CLI

```bash
python -m pytest tests/test_label_builder.py tests/test_rolling_split.py

```

## Boundaries

This phase prepares model-ready artifacts but does not claim formal predictive
results. Large raw filings, licensed transcripts, CRSP/Compustat extracts, and
other restricted datasets should not be committed to GitHub.
