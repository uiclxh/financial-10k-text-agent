# v0.6.0 Step 14 - Complete Local MVP Deployment

Release date: 2026-05-21

This release completes Step 14 for the local MVP: the project now has an
executable local orchestration path, configured input locations, raw local 10-K
parsing support, reproducibility documentation, and GitHub CI smoke deployment.

## Phase

Step 14:

- Complete local MVP orchestrator
- Configured local artifact inputs
- Raw local 10-K parsing orchestration
- GitHub Actions CI
- Dockerfile and Makefile
- End-to-end smoke fixture run
- Deployment and reproducibility guide

## Deployment Placement

Local first, GitHub for source code, tests, CI, and small fixtures.

Generated run artifacts, licensed filings, raw price data, model caches, and
large reports remain local by default.

## Included Capabilities

- `inputs` config block:
  - `document_manifest_path`
  - `prices_path`
  - `parsed_sections_path`
  - `raw_filings_dir`
  - `copy_inputs_to_run_dir`
- `python -m text_factor_lab run --config <path> --execute` now:
  - copies configured input artifacts into the run directory when requested
  - blocks immediately when configured inputs are missing
  - parses local raw 10-K files when `parsed_sections.jsonl` is absent
  - writes `parsing_quality_report.json`
  - continues through features, models, evaluation, audit, and report
- `pipeline_contract.json` documents the complete local MVP controller.
- `orchestrator_report.json` uses `artifact-aware-orchestrator-v1`.
- `.github/workflows/tests.yml` runs lint and tests in GitHub Actions.
- `configs/text_factor_lab/e2e_smoke.yaml` and `examples/e2e_smoke/` provide a
  tiny committed end-to-end smoke run.
- `Dockerfile` and `Makefile` provide local reproducibility commands.
- `docs/deployment.md` documents setup, artifact policy, run modes, and
  deployment boundaries.

## Boundaries

Step 14 completes the local deployable MVP, not a production cloud service and
not a claim of formal research-grade empirical results. Formal runs still
require research-grade universe construction, licensed data coverage, benchmark
quality control, and stronger portfolio time-series backtesting.
