# v0.6.0 Step 14 - Complete Local MVP Deployment

Release date: 2026-05-21

Complete local MVP deployment release. This version adds configured input
locations, local raw 10-K parsing orchestration, CI smoke validation,
reproducibility packaging, and a tiny end-to-end demo run.

## Changes

- Added `inputs` config block:
  - `document_manifest_path`
  - `prices_path`
  - `parsed_sections_path`
  - `raw_filings_dir`
  - `copy_inputs_to_run_dir`
- Extended the executable run controller so it can:
  - copy configured input artifacts into the run directory
  - block immediately when configured inputs are missing
  - parse local raw 10-K files when `parsed_sections.jsonl` is absent
  - write `parsing_quality_report.json`
  - continue through features, models, evaluation, audit, and report
- Updated `pipeline_contract.json` for the complete local MVP controller.
- Updated `orchestrator_report.json` to `artifact-aware-orchestrator-v1`.
- Added GitHub Actions CI for lint and tests.
- Added `configs/text_factor_lab/e2e_smoke.yaml`.
- Added `examples/e2e_smoke/` for tiny committed end-to-end validation.
- Added `Dockerfile`.
- Added `Makefile`.
- Added `docs/deployment.md`.

## Boundaries

Step 14 completes the local deployable MVP, not a production cloud service and
not a claim of formal research-grade empirical results. Formal runs still
require research-grade universe construction, licensed data coverage, benchmark
quality control, and stronger portfolio time-series backtesting.
