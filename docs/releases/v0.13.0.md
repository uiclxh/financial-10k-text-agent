# v0.13.0 Daily Portfolio

Release date: 2026-05-23

Portfolio-simulation upgrade for daily price-driven return series. This release
keeps the original event-window label return fallback while allowing the
portfolio layer to compute daily long-short returns from the price panel.

## Changes

- Added optional price-panel input to evaluation artifact construction.
- Added `--prices` to `evaluate-models`.
- Added `--portfolio-return-type` to `evaluate-models`.
- Added automatic daily portfolio simulation in `run --execute` when run-dir
  `prices.csv` is available.
- Added `return_source` to `PortfolioReturnRecord`.
- Marked legacy event-window portfolio returns as `label_window`.
- Marked price-driven daily portfolio returns as `daily_price_panel`.
- Applied transaction costs on the first holding date after a rebalance.
- Preserved existing equal-weight, value-weight, sector-neutral equal-weight,
  and sector-neutral value-weight portfolio variants.
- Added report summary tracking for portfolio return sources.
- Added tests for daily price-driven return construction.
- Added e2e smoke assertion that portfolio returns use `daily_price_panel` when
  prices are provided.

## Boundaries

This release computes daily returns from fixed rebalance weights over each
holding window. It does not yet implement full position drift, overlapping
sub-portfolio accounting, borrow costs, capacity constraints, or execution
slippage.
