# v0.3.0 Phase 3 - Models, Evaluation, Backtest, Audit

Release date: 2026-05-20

This release packages the current runnable MVP research loop. Formal large-sample
experiments should still run locally, but GitHub should contain the tested code,
small fixtures, smoke tests, and release documentation.

## Phase

Steps 9-12:

- Model training
- Out-of-sample evaluation
- Event-based long-short backtest
- Audit gate

## Deployment Placement

Local + GitHub CI smoke test.

## Why This Phase Belongs Here

Formal experiments can be large and should run locally first. Small deterministic
tests belong on GitHub so the model, evaluation, backtest, and audit contracts
stay reproducible. This is the right phase for CI smoke tests because the main
research loop now has enough executable surface area to catch regressions.

## Included Capabilities

- Model layer:
  - `historical_mean`
  - `industry_mean`
  - `ridge`
  - optional `xgboost`
- Validation-safe model-selection flow:
  - validation predictions come from train-fit models
  - test predictions come from train+validation refit models
- `predictions.jsonl`.
- `model_manifest.json`.
- `tuning_log.json`.
- Evaluation metrics:
  - RMSE
  - MAE
  - R2
  - directional accuracy
  - Pearson IC
  - rank IC
  - split-level and `ALL_SPLITS` aggregate records
- Event-based long-short backtest:
  - top/bottom quintile with minimum one name per leg
  - equal-weight prototype returns
  - transaction-cost adjustment
  - Newey-West t-stat
  - Sharpe ratio
- Audit gate:
  - artifact presence
  - schema validation
  - prediction-label alignment
  - coverage threshold
  - split leakage failures
  - feature lookahead
  - TF-IDF train-window fit-scope
  - feature manifest / vocabulary alignment
  - validation-only tuning metric check
  - tested-specifications disclosure
  - formal metadata gate

## Boundaries

This is still an MVP research loop. It does not yet include full one-command
orchestration, sector-neutral portfolio construction, value-weighted portfolios,
real portfolio time-series turnover, multiple-testing adjustment reports, or the
final report agent. Those belong to the planned Step 13-14 release.
